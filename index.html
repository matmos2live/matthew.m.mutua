Binary Exploitation: Buffer Overflow

Problem Statement:

Exploiting a buffer overflow vulnerability in a C program to gain control of execution flow and achieve shell access.

Approach:

Used GDB to analyze the program's memory layout and identify the vulnerable buffer. Crafted a payload with NOP sled, shellcode, and return address overwrite to point to shellcode.

Tools Used:

GDB pwntools (Python library) objdump

Screenshots:

Key Lessons Learned:

  • Deep understanding of stack frames, function calls, and return addresses.
  • Techniques for crafting precise exploit payloads.
  • Mitigation techniques like ASLR and DEP and how they complicate exploitation.
